## 1. Как устроен проект (общая база — знают все)

Реал-тайм рендер подводного мира на **OpenGL 3.3 core**, без игровых движков — рендер,
свет, вода, частицы и игрок написаны руками.

Поток управления:

- `main.cpp` создаёт окно (GLFW), GL-контекст, ImGui-панель, читает ввод.
- `PlayerController` — ввод W/A/S/D + мышь → quaternion-камера.
- `Scene::Update()` — обновляет все подсистемы.
- `Scene::Render()` — рисует кадр (7 проходов ниже).

**7 проходов кадра** (порядок в `Scene_Render.cpp`):

1. cpu/update — системы, физика игрока, анимации.
2. shadow + caustics — depth-проход от солнца → shadow map; каустика в текстуру.
3. **terrain** — дно из heightmap (PBR + нормал-маппинг) — *Микита*.
4. **vegetation** — тысячи растений инстансингом + LOD — *Саша*.
5. **props + существа** — пропсы (LOD) + анимированные твари — *Саша*.
6. **water** — поверхность воды (отражение неба) + прозрачность/преломление (видно дно сквозь воду) — *Артём*.
7. **post** — god rays, туман, тонмаппинг → экран — *Артём*.

Частый вопрос: **«почему вода после террейна?»** → прозрачность/преломление воды читают
**уже готовую** непрозрачную сцену (видно дно сквозь воду).

### 📚 Учить

**`scene/Scene_Terrain.cpp` → `createTerrainFromHeightmap()` — ядро B07 (стр. 139-537)**

- Грузит PNG-heightmap (стр. 141), строит сетку 1024×1024 (стр. 161).

### ❓ 10 вопросов, которые спросят (с ответами)

1. **Как у вас меняется день/ночь?** — Слайдер `Time of Day` → угол солнца (`main.cpp:425-426`) → одно направление света `sunDir` гонится во все шейдеры (тени, цвет неба, god rays).
2. **Откуда берётся рельеф дна?** — Из PNG-heightmap: яркость пикселя = высота, грузит `HeightmapLoader`.
3. **Как сглаживаете высоту между пикселями?** — Бикубическим **Catmull-Rom** сэмплом.
4. **Как считаете нормали террейна?** — **Sobel 3×3** по высотам соседей (built-in low-pass, меньше фейсетинга).
5. **Что такое normal mapping и где TBN?** — Нормаль из карты задана в tangent-space; матрицей **TBN** переводим в мир. Делается в `terrain.frag`.
6. **На скольких материалах нормал-маппинг?** (требование ≥2) — Террейн (triplanar) + пропсы/вода. Минимум выполнен.
7. **Что такое PBR metallic/roughness?** — Материал = albedo + metallic + roughness; specular = **D·G·F** (GGX × Smith × Fresnel).
8. **Как работает shadow mapping?** — Рендерим глубину из позиции солнца, в основном проходе сравниваем глубину фрагмента с картой.
9. **Что такое shadow acne и как лечите?** — Самозатенение из-за точности → **bias** + усреднение **PCF** (3×3).
10. **Зачем дно разбито на чанки?** — Чтобы рисовать только видимые тайлы (frustum culling) и брать дальним LOD.

### ❓ 10 вопросов, которые спросят (с ответами)

1. **Что такое gimbal lock и почему у вас его нет?** — При углах Эйлера на полюсе оси совпадают, теряется степень свободы. Кватернион хранит ось+угол — блокировки нет.
2. **Как мышь крутит камеру?** — Двумя кватернионами: yaw вокруг мировой оси (горизонт ровный) + pitch вокруг локального Right, потом композиция.
3. **Почему кватернион, а не матрица Эйлера?** — Нет накопления перекоса, плавная интерполяция, базис берётся прямо из кватерниона.
4. **Чем сэмплится кубмапа?** — Направлением взгляда (vec3), не UV.
5. **Почему кубмапа не едет за камерой?** — Из view-матрицы убираем трансляцию (берём mat3), остаётся только поворот.
6. **Какой порядок проходов кадра?** — 7 пассов: тени/каустика → террейн → растительность → пропсы → вода → пост.
7. **Почему вода рисуется последней?** — Прозрачность/преломление воды читают уже готовую непрозрачную сцену (видно дно сквозь воду) и блендятся поверх.
8. **Что такое волны Гёрстнера?** — Смещение вершин суммой нескольких волн + пересчёт нормалей для бликов.
9. **Forward или deferred рендер?** — Forward.
10. **Что в пост-обработке?** — God rays (радиальный blur от экранной позиции солнца), туман глубины, тонмаппинг.

# 6. 👤 САША — инстансинг, растительность, существа

**Защищаю:** 🟩 A07 + Parallel Transport Frames.
**Демо:** 🐟 рыбы уплывают от ныряльщика.

### 📚 Учить

**`scene/VegetationSystem.cpp` — A07-эталон (587 стр.)**

- `buildLOD()` (стр. 30-124) — 3 процедурные LOD-сетки (запасной вариант для морской травы; кораллы/келп заменяются `loadModel()`). Каждая карта = `segments` квадов = 2·segments трисов; в `init()` (130-132): **LOD0 = 4 карты×6 = 48 трис, LOD1 = 2×4 = 16, LOD2 = 1×2 = 4** (точные числа печатаются на старте: «Vegetation LODs built: …»).

- `scatter()` (стр. 338-424) — раскидка по heightmap (фильтры slope/band/noise); каждому даётся `lifeDist` (стр. 416) — личный радиус исчезновения.

- 🛡️ *«per-instance VBO + divisor=1 + glDrawElementsInstanced; 3 LOD по дистанции; стохастический pruning — нет видимой стены».*

**`scene/SplinePath.h` — Parallel Transport Frames (200 стр.)**

- PTF-цикл (стр. 102-120): нормаль переносится поворотом на угол между касательными (`axis = T0×T1`, `angle = acos(T0·T1)`) → нет твиста.

**`scene/Scene_Props.cpp` (554 стр.)** — те же идеи для пропсов: дискретный LOD + frustum culling. Глянуть ключевое.

### ❓ 10 вопросов, которые спросят (с ответами)

1. **Что такое инстансинг?** — Один меш рисуется много раз за **1 draw call**; per-instance данные (позиция/поворот/масштаб) в отдельном VBO.
2. **Что делает `glVertexAttribDivisor(…, 1)`?** — Атрибут меняется **раз на инстанс**, а не на каждую вершину. Без этого инстансинг не работает.
3. **Сколько у вас LOD и как выбирается?** — 3 уровня (96/32/4 трис), выбор по расстоянию до камеры (`lodDist0/lodDist1`).
4. **Что такое LOD popping и как боретесь?** — Резкий скачок при смене уровня; смягчаем **стохастическим прунингом** (каждый куст гаснет на своём радиусе) + дискретные уровни.
5. **Сколько draw call на всю растительность?** — По одному `glDrawElementsInstanced` на каждый из 3 LOD-бакетов.
6. **Что такое Parallel Transport Frames?** — Рамка ориентации, переносимая вдоль кривой минимальным поворотом → труба/змей не закручивается.
7. **Чем PTF отличается от рамки Френе?** — Френе строится на кривизне и скачет на прямых/перегибах; PTF переносит рамку минимальным поворотом — твиста нет.
8. **Как рыбы реагируют на игрока?** — Уплывают (flee) в радиусе 8 м, отталкивание 1/r — сильнее вблизи (`FishSchool.cpp:151-158`).
9. **Как растения качаются?** — Деформация в vertex-шейдере (sinus по высоте и времени), без пересчёта на CPU.
10. **Чем грузите модели?** — Через Assimp (`ModelLoader`), OBJ/glTF → внутренний `LoadedMesh`.
