## 1. Как устроен проект (общая база — знают все трое)

Реал-тайм рендер подводного мира на **OpenGL 3.3 core**, без игровых движков —
рендер, свет, вода, частицы и игрок написаны руками.

**Поток управления:**
```
main.cpp
  ├─ окно (GLFW) + GL-контекст + ImGui-панель + ввод
  ├─ PlayerController   (W/A/S/D, мышь → quaternion-камера)
  ├─ Scene::Update()    обновляет все подсистемы
  └─ Scene::Render()    рисует кадр (7 проходов ↓)
```

**7 проходов кадра** (буквально порядок в `Scene_Render.cpp`):
```
[1] cpu/update        системы, физика игрока, анимации
[2] shadow + caustics depth-проход от солнца → shadow map; каустика в текстуру
[3] terrain           дно из heightmap (PBR + нормал-маппинг)   ← Микита
[4] vegetation        тысячи растений инстансингом + LOD        ← Саша
[5] props + существа  пропсы (LOD) + анимированные твари        ← Саша
[6] water + ssr       поверхность воды + отражения              ← Артём
[7] post              god rays, туман, тонмаппинг → экран        ← Артём
```
Частый вопрос на защите: **«почему вода после террейна?»** → SSR/прозрачность читают
**уже готовую** непрозрачную сцену.

## 10. Вероятные вопросы на защите (теория по методам)

Коротко: вопрос → что ответить. Зубрить не надо — понять суть.

### Обязательные

**Quaternion-камера**
- *Что такое gimbal lock?* — При углах Эйлера на «полюсе» две оси совпадают → теряется
  степень свободы, камера дёргается/кувыркается. Кватернион хранит ось+угол — блокировки нет.
- *Почему кватернион, а не матрица Эйлера?* — Нет накопления перекоса, плавная интерполяция,
  компактнее. Базис берём прямо из кватерниона.

**Cubemap**
- *Чем сэмплится?* — Направлением взгляда (vec3), не UV-координатами.
- *Зачем?* — Фон/окружение/отражения; у нас процедурный подводный куб, открывается при погружении.

**Normal mapping**
- *Что такое tangent space / TBN?* — Локальный базис (Tangent, Bitangent, Normal) на поверхности;
  нормаль из карты задана в нём → переводим в мир матрицей TBN.
- *Зачем triplanar на террейне?* — Нет UV-швов и растяжений: текстура проецируется по 3 осям и блендится.

**PBR (Cook-Torrance)**
- *Что такое BRDF?* — Функция, как свет отражается от поверхности (вход: L, V, N → выход: яркость).
- *Из чего specular?* — `D·G·F / (4·NdotL·NdotV)`: **D** = GGX (распределение микрограней),
  **G** = Smith (самозатенение микрограней), **F** = Fresnel-Schlick (рост блика на скользящем угле).
- *Roughness / metallic?* — roughness = ширина блика; metallic → базовая отражательность F0
  (диэлектрик ~0.04, металл = albedo).
- *Энергосбережение?* — `kd = (1 - F)·(1 - metallic)`, диффуз + спекуляр не дают больше, чем пришло.

**Shadow mapping + PCF**
- *Как работает?* — Рендерим сцену из позиции солнца в depth-текстуру; в основном проходе
  переводим фрагмент в свет-пространство и сравниваем его глубину с сохранённой.
- *Shadow acne / peter-panning?* — Самозатенение из-за точности → лечим **bias**.
- *Что даёт PCF?* — Усредняем N соседних тапов (у нас 3×3=9) → мягкий край вместо рваного.

**Parallel Transport Frames**
- *Чем отличается от рамки Френе?* — Френе строится на кривизне → нормаль скачет на прямых
  участках и крутит сечение. PTF переносит рамку **минимальным поворотом** → трубу не закручивает.

### Оцениваемые

**B07 — Heightmap seabed**
- *Как из картинки рельеф?* — Яркость пикселя = высота; между текселями интерполируем
  (Catmull-Rom), нормали считаем из соседей (Sobel 3×3).
- *Зачем чанки и AABB?* — Frustum culling (рисуем только видимые тайлы) + LOD по расстоянию.

**A07 — Instancing + LOD**
- *Что такое инстансинг?* — Один меш рисуется много раз за **1 draw call**; per-instance данные
  (позиция/поворот/масштаб) лежат в отдельном VBO с `glVertexAttribDivisor=1`.
- *Зачем LOD?* — Дальние растения = меньше треугольников (у нас 96/32/4 на 3 уровнях).
- *Что такое LOD popping и как боретесь?* — Резкий «скачок» при смене уровня; смягчаем
  **стохастическим прунингом** (каждый куст гаснет на своём радиусе) + дискретные уровни по дистанции.

### Общее по пайплайну
- *Forward или deferred рендер?* — **Forward**.
- *Порядок кадра?* — 7 проходов (п.1): тени/каустика → террейн → растительность → пропсы/твари → вода+SSR → пост.
- *Почему вода/прозрачность в конце?* — SSR читает уже готовую непрозрачную сцену, blend кладётся поверх.
- *Координаты:* model → world → view → clip (MVP); нормали — через normal matrix.
